## Who We Are

Zero One Creative is a startup building **Amara** — a suite of AI-powered 3D creative tools that make professional-grade 3D content creation accessible to everyone. Our small team of ML engineers, full-stack developers, and product-minded founders works fast, ships often, and cares deeply about craft.

We believe the best tools disappear into the creative process. Amara should feel like an extension of the artist's mind, not a technical hurdle to overcome.

## What You'll Be Working On

Amara is composed of several interconnected systems:

- **Amara Studio** — the web-based 3D editor where users create and manipulate scenes
- **Amara Engine** — the ML inference pipeline that powers AI-assisted generation
- **Amara API** — the backend services connecting the editor to the engine
- **Amara Marketplace** — a platform for sharing and discovering 3D assets

Depending on your role, you'll touch one or more of these systems daily.

## Your First Week

### Day 1: Access & Environment

Get your accounts set up and your development environment running. You should have received invitations to:

- **GitHub** — our code lives in the `zero-one-creative` organisation
- **Jira** — sprint planning and task tracking
- **Slack** — daily communication (see the Communication Norms policy)
- **Notion** — design docs and meeting notes (being migrated here)
- **Vercel** — deployment dashboard

> **Tip:** If you're missing access to anything, message #team-ops in Slack. Don't waste time waiting — we want you productive on day one.

### Day 2–3: Codebase Orientation

Clone the main repositories and get them running locally. The README in each repo has setup instructions. Start with the repo most relevant to your role:

- **ML engineers** → `amara-engine`
- **Full-stack** → `amara-studio` and `amara-api`

Read through the PR Process SOP and Branch Naming Reference before making your first commit.

### Day 4–5: First Contribution

Pick up a starter task from the "Good First Issue" column in Jira. These are intentionally small and well-scoped — the goal is to get you through the full workflow (branch, code, PR, review, merge) so the process is familiar before you tackle bigger work.

## How We Work

We run **two-week sprints** with planning on Mondays and retros on alternate Fridays. Daily standups happen async in Slack (#daily-standup) — post what you did yesterday, what you're doing today, and any blockers.

We value:

- **Shipping over perfection** — get it in front of users, then iterate
- **Clear communication** — write things down, explain your reasoning
- **Ownership** — if you see a problem, you have permission to fix it
- **Kindness** — we're a small team and we look after each other
